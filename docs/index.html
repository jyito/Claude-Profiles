One account at a time is not enough

Personal plan, work plan, a client's account — Claude Desktop signs in one at a time, so you're forever logging out and back in.

Claude Profiles gives every account its own app icon — Claude Personal, Claude Work, Claude Client X — each with a distinct colored badge so you can tell them apart at a glance. Open as many as you like, all at once. Each stays signed in forever.

How it works — and why it's safe

It's a thin, honest launcher around the real Claude Desktop app.

Claude Desktop is an Electron app. Launched with --user-data-dir=<path>, it keeps all of its session state inside that folder. Claude Profiles simply gives each account its own folder and a tiny native wrapper that launches the real app pointed at it — exactly like separate browser profiles.

No credential handling, ever. It never sees, stores, or transmits your passwords or tokens. Claude Desktop manages its own session in each folder.
No telemetry, no network. Nothing phones home. The only "analytics" is a local launch-count file you can delete.
Never modifies Claude.app. The official app's signature stays intact; its auto-updates keep working.
Zero dependencies. Plain macOS built-ins — no Homebrew, no Node, no Python, no compilation.

A real app, not a script

Everything you'd want to manage a fleet of Claudes.

Live dashboard

Native window showing each instance's CPU, memory, terminals, and disk, with rolling sparklines — refreshed every 2 seconds.

Per-profile badges

Each account gets a distinct colored badge on its icon, so a Dock full of Claudes is instantly readable. Pick the color yourself.

Drill-down & cleanup

Expand a card to see live terminal sessions and close idle ones, or clear regenerable caches by tier — never your logins.

Show Window

One click raises a specific instance's windows by process, even though all instances share Claude's bundle — no special permissions.

Throttle & auto-clean

Tame a CPU-hungry instance, or opt into automatic cache cleanup — all off by default, all local.

Safe by design

Removing a profile keeps your login unless you type DELETE. An Emergency Stop is one click away. Sign-ins always survive.

Build it in three commands

macOS only. No compiler, no dependencies.

# clone, build, run
git clone https://github.com/jyito/Claude-Profiles.git
cd Claude-Profiles
bash scripts/build.sh && open "dist/Claude Profiles.app"

Or grab a release build. First launch of a downloaded app is right-click → Open (it's unsigned for now). Profiles the app creates locally open normally.

Questions

Is it really free?

Yes — Apache-2.0 licensed, source on GitHub, no subscription, no upsell, no account required. You should not have to pay for something this simple. That's the whole point.

Is it safe with my logins?

It never touches your credentials. Each account's session lives in its own folder, managed by Claude Desktop itself — the same mechanism browsers use for profiles. No network, no telemetry.

Is this an official Anthropic product?

No. It's an unofficial, independent community tool. It doesn't modify Claude Desktop and isn't affiliated with or endorsed by Anthropic.

Why do running windows still say "Claude"?

Because each profile launches the real, unmodified Claude.app — which is exactly what keeps it safe and auto-updating. The per-profile badge and Show Window tell instances apart instead.
